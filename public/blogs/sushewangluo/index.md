

## 前言：技术源于生活痛点

作为一名热衷于“Vibe Coding”和折腾各种硬件的大学生，宿舍的网络环境一直是我关注的重点。起因很简单：宿舍的原始网络往往存在设备连接数受限、特定端口被封禁、以及内网互访困难等问题，甚至有时还要忍受室友深夜打游戏时“不可控”的网络资源抢占和噪音干扰。

为了彻底掌控宿舍的网络环境，为我的**机械臂开发（内网通信）**和各种 IoT 设备提供一个稳定、自由的基础设施，我决定对宿舍网络进行一次“大手术”——构建一套基于**硬主路由 + 软旁路由**的混合网络架构，并利用 Kali Linux 进行了一些基础的网络安全与渗透测试探索。![](https://i.imgs.ovh/2026/04/29/jIORrX.webp)

## 第一阶段：网络架构重构——硬主软旁，各司其职![](https://i.imgs.ovh/2026/04/29/jIyuUO.webp)

传统的单级路由器方案在面对高并发或复杂规则时往往捉襟见肘。我的思路是：让专业的设备干专业的事。

- **硬件主路由（稳定至上）：** 负责基础的拨号、NAT 转发和无线 AP 功能，确保整个宿舍“能上网”，保证网络的最底线稳定性。
    
- **软旁路由（功能核心）：** 负责复杂的网络包处理，比如去广告、特定流量转发、自定义 DNS 解析等高级功能。
    

### 实战步骤：小米路由器 3G 的“浴火重生”
![](https://i.imgs.ovh/2026/04/29/jIOVUN.webp)
1. **刷入 Breed（不死控制台）：** 这是折腾路由器的第一步。我通过 SSH 登录到路由器后台，利用漏洞刷入了 Breed。这相当于给路由器装了一个底层系统，以后无论怎么刷固件都不怕变砖。
    
2. **编译与刷入 LEDE/OpenWrt：** 我没有选择现成的臃肿固件，而是根据自己的需求（精简核心+特定插件），编译了 LEDE (OpenWrt 的分支) 固件。![](https://i.imgs.ovh/2026/04/29/jIOLo4.png)
    
3. **旁路由（Bypass Gateway）配置：** 这是架构的核心。
    
    - **主路由配置：** 关闭主路由的 DHCP 功能，或者将主路由 DHCP 分配的网关和 DNS 指向旁路由的 IP。
        
    - **旁路由配置：** 将软路由的 IP 设置在主路由同一网段，并开启 DHCP 服务（或者利用主路由 DHCP 引导）。关键在于关闭旁路由的 DHCP 动态伪装（防止 NAT 冲突），并设置自定义的防火墙规则：`iptables -t nat -I POSTROUTING -o eth0 -j MASQUERADE`，确保流量能正确回传给主路由。
        

**架构收益：**

现在，我的机械臂“LANTERN”和所有测试设备都能在同一个纯净的内网中高速通信，同时所有的出海流量或特定请求都由软路由接管处理，做到了无感知的网络优化。

## 第二阶段：从网络工程到网络安全——Kali Linux 的试炼

网络架构搭建完毕后，我意识到“掌控网络”不仅是能连网，更包括了解网络中的数据流向和安全隐患。由于宿舍环境偶尔会出现一些“不和谐”的干扰因素（比如严重影响休息的噪音），我决定利用这套网络架构，在授权范围内（自己的设备与网络）进行一些安全测试。

### 1. ARP 欺骗与流量分析（理论验证）

为了理解局域网内的设备是如何互相发现和通信的，我搭建了 Kali Linux 环境。

- **原理：** 利用 ARP 协议的无状态特性，向目标设备发送伪造的 ARP 响应包，告诉它“我就是网关”。
    
- **实践：** 使用 `arpspoof` 工具。如果操作成功，目标设备的流量就会先经过我的电脑，再转发给真正的网关。这让我深刻理解了局域网中间人攻击（MITM）的原理，也明白了为什么在公共 Wi-Fi 下不应该传输敏感信息。
    

### 2. “幽灵攻击”：一种技术化的“善意提醒”

面对宿舍噪音问题，我没有选择无休止的争吵，而是尝试用技术手段进行了一次“降维打击”。

- **局域网控制：** 结合软路由的 Mac 地址绑定和防火墙规则，我编写了自动化脚本。当环境分贝检测器（我自己魔改的一个小工具）连续超过设定阈值时，脚本会自动通过 SSH 连接到路由器。
    
- **定向断网/降速：** 脚本会执行预设的 iptables 规则，对特定设备的 IP/Mac 进行短暂的断网或极速限流。
    
- **效果：** 这是一种无需接触物理设备的“幽灵”操作。在测试了几次后，对方意识到了网络的不稳定与特定行为之间的关联。这不仅解决了我个人的困扰，更让我验证了网络层控制硬件行为的有效性。
    

_（注：此部分操作仅在个人局域网实验环境中进行，核心目的是为了验证网络协议的脆弱性和自动化控制的逻辑，并非鼓励恶意攻击。）_

## 总结：向智能制造迈进的基础

通过这次“硬主软旁”的网络改造和基于 Kali 的安全探索，我收获的不仅仅是一个好用的宿舍 Wi-Fi。

1. **底层逻辑的理解：** 我深入理解了 IP/MAC 地址、网关、NAT、路由表以及 ARP 协议的实际工作方式。
    
2. **软硬协同的能力：** 智能制造不仅仅是机械结构，更是设备与设备之间的协同。这种搭建底层通信网络的能力，是未来工业互联网（IIoT）的基础。
    
3. **问题导向的工程思维：** 遇到问题（网络受限/噪音干扰），分析痛点，查阅资料，动手实践，最终用技术手段解决。这正是我一直推崇的“工程师文化”。
    

目前的网络架构还比较初级，下一步，我计划将我的 机械臂彻底接入这个内网，开发一套基于 Web 的远程控制“上位机”面板，让“网络工程”真正服务于“机械控制”。
